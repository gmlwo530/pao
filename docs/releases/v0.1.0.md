# PAO v0.1.0 Release Notes

PAO v0.1.0 is the first usable development release of Project Agent Orchestrator, a macOS-first terminal AI coding agent for coordinating local AI coding CLIs across multiple Git repositories.

## Highlights

- Initialize a PAO workspace with `pao init`.
- Register, list, inspect, remove, and synchronize Git repositories in a workspace.
- Register local AI client commands and choose a default client.
- Create task storage under `.pao/tasks/<task-id>/`.
- Prepare `pao chat --repo <name> --prompt <prompt>` approval artifacts before AI client execution.
- Capture baseline repository state and classify changed, untracked, deleted, overlapping, and outside-repo paths.
- Model approval actions, risk levels, command log entries, and redaction boundaries.
- Run `pao doctor` to inspect local tooling, workspace, repository, config, and client command readiness.
- Render stable PAO error codes with human-readable messages.

## Installation

Install from a local checkout:

```bash
cargo install --path .
```

## Basic Usage

```bash
pao init
pao repo add app --remote <git-url> --branch main
pao client add codex --command codex
pao chat --repo app --prompt "make a small change"
pao doctor
```

## Verification

The release branch should pass:

```bash
cargo fmt --check
cargo check
cargo test
cargo clippy --all-targets -- -D warnings
```

## Current Limits

- The default `pao` TUI entrypoint is not implemented in v0.1.0.
- `pao chat` prepares baseline and approval artifacts; AI client execution remains a follow-up v0 path.
- v0 command, configuration, and workspace contracts may change before `v1.0.0`.
